# CEFANET Digital Notice Board — Deployment Guide

This is the **end-to-end click-by-click** for putting the demo on the public internet. Total time: **~25 minutes**. All three services are free tier — no credit card needed.

## The hosting trio

```
┌───────────────────┐        ┌──────────────────┐        ┌───────────────────┐
│  Browser (user)   │ ─────▶ │   Vercel         │ ─────▶ │   Render          │
│                   │        │   Next.js 14     │        │   Express API     │
│                   │        │   cefanet.       │        │   cefanet-api.    │
│                   │        │   vercel.app     │        │   onrender.com    │
└───────────────────┘        └──────────────────┘        └──────────┬────────┘
                                                                    │
                                                                    ▼
                                                          ┌─────────────────┐
                                                          │  Neon           │
                                                          │  Postgres 16    │
                                                          │  *.neon.tech    │
                                                          └─────────────────┘
```

| Component | Service | Cost | Why |
|---|---|---|---|
| Database | **Neon** | Free tier | Serverless Postgres, sleeps when idle, zero cost |
| API (Express) | **Render** | Free tier | Long-running Node — perfect for Express |
| Web (Next.js) | **Vercel** | Free tier | Built by Next.js's authors — zero config |

> The Render free tier **sleeps after 15 min of no traffic** and takes ~30 sec to wake. For a live demo, hit the API once a few minutes before you start ("warming it up").

## Step 2 — Deploy the API to Render (10 min)

1. Go to https://render.com → **Get Started** → sign in with GitHub.
2. Render dashboard → **New** → **Blueprint**.
3. Connect your `cefanet-dnb` repo. Render detects the `render.yaml` and proposes one service: **cefanet-api**.
4. Click **Apply**. Render starts the first build.
5. While it's building, Render will prompt you to fill in the two `sync: false` secrets:

   | Variable | Value |
   |---|---|
   | `DATABASE_URL` | Paste your full Neon connection string (the same one in your local `.env`, ending in `?sslmode=require`) |
   | `CORS_ORIGIN` | Leave empty for now — you'll come back after the Vercel deploy |

   `JWT_SECRET` is auto-generated by Render — leave it.

6. Wait for the build to finish (~3 min). You'll see "Live" in green when ready.
7. Copy the service URL — something like **`https://cefanet-api.onrender.com`**. Test it:

   ```
   https://cefanet-api.onrender.com/health
   ```

   Should return JSON: `{"ok":true,"service":"cefanet-api",...}`.

## Step 3 — Deploy the Web app to Vercel (8 min)

1. Go to https://vercel.com → **Sign Up** with GitHub.
2. Dashboard → **Add New** → **Project**.
3. Pick the `cefanet-dnb` repo → **Import**.
4. **Critical setting** — under "Configure Project":
   - **Root Directory** → click **Edit** → select `apps/web`
   - Framework Preset: **Next.js** (auto-detected)
   - Build Command, Output Directory, Install Command — **leave the defaults**, the `vercel.json` handles everything
5. Expand **Environment Variables** and add:

   | Name | Value |
   |---|---|
   | `NEXT_PUBLIC_API_URL` | The Render URL from Step 2, e.g. `https://cefanet-api.onrender.com` |

6. Click **Deploy**. Wait ~2 min. You'll get a URL like **`https://cefanet-dnb-xxx.vercel.app`**.

---

## Step 4 — Wire CORS back to the API (2 min)

The API doesn't know to trust your Vercel domain yet — so all login/dashboard calls will be blocked.

1. Render dashboard → your `cefanet-api` service → **Environment**.
2. Edit `CORS_ORIGIN` and set it to your Vercel URL (no trailing slash):

   ```
   CORS_ORIGIN=https://cefanet-dnb-xxx.vercel.app
   ```

3. Save. Render auto-redeploys (~30 sec).

If you have multiple Vercel domains (preview + prod), comma-separate:
```
CORS_ORIGIN=https://cefanet-dnb-xxx.vercel.app,https://cefanet-dnb-git-main-yourname.vercel.app
```

## Step 6 — (Optional) Custom domain

Both Vercel and Render let you add a custom domain on the free tier.

**Vercel** → Project Settings → Domains → Add → e.g. `cefanet.org`. Follow the DNS instructions Vercel gives you.

**Render** → Service Settings → Custom Domain → e.g. `api.cefanet.org`. Then update `NEXT_PUBLIC_API_URL` on Vercel and `CORS_ORIGIN` on Render to match.

## Common issues during deployment

| Symptom | Cause | Fix |
|---|---|---|
| Render build fails with "Cannot find module 'tsx'" | tsx was in devDependencies | Already fixed — `tsx` is now in `dependencies` of `apps/api/package.json` |
| Vercel build fails: "Cannot find module '@cefanet/shared'" | Wrong root directory | In Vercel project settings → set Root Directory to `apps/web` |
| Browser shows "Failed to fetch" on login | CORS_ORIGIN doesn't match the exact Vercel URL | Copy the URL from your browser address bar (no trailing slash) into Render's `CORS_ORIGIN` |
| API returns 500 with PostgresError | DATABASE_URL not set or wrong | Render Environment → confirm Neon URL ends in `?sslmode=require` |
| Login button does nothing | Stale browser cache | Hard-reload: `Ctrl+Shift+R` |
| Render says "your service is sleeping" | 15-min idle on free tier | Visit `https://<api>.onrender.com/health` once to wake it — takes ~30 sec |

## Re-deploying after code changes

After this initial setup, everything is automatic:

1. Commit changes locally:
   ```powershell
   git add .
   git commit -m "Your change description"
   git push
   ```
2. Vercel rebuilds the web automatically (~1 min).
3. Render rebuilds the API automatically (~3 min).

You'll see deploy status in both dashboards.

## Rotating the demo credentials

**Database password** (Neon):
1. Neon console → your project → **Roles** → `neondb_owner` → **Reset password**
2. Copy the new connection string
3. Update `DATABASE_URL` in both: local `.env` AND Render dashboard
4. Render redeploys automatically

**JWT secret** (Render):
1. Render dashboard → Environment → edit `JWT_SECRET` → click "Generate"
2. Save. All existing user sessions are invalidated (forces re-login).

**Demo user passwords** (in the seeded data):
The seed script hard-codes `demo123` for both demo accounts. To change, edit `scripts/seed.ts` and re-run `npm run seed` against your Neon DB.
